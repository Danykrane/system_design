# 09. Надежность — сценарии и обоснования

## Базовые механизмы

| Механизм | Где используется | Цель |
|---|---|---|
| N+1 per pool | API, SFU, LB | пережить отказ узла без остановки |
| RF=3 + quorum | stateful контуры | корректный failover |
| Retry + exponential backoff | клиент и межсервисные вызовы | устойчивость к кратким сбоям |
| Circuit breaker | межсервисные вызовы | защита от каскадной деградации |
| Graceful shutdown | все stateless сервисы | уменьшение потерь запросов/сессий |

## Degradation policy

| Сбой | Что отключаем первым | Что сохраняем |
|---|---|---|
| рост ошибок/latency в media plane | HD/1080p и запись | audio + SD video |
| перегруз API | тяжелые не-критичные endpoint | auth и signaling |
| проблемы аналитики | async events processing | основной пользовательский поток |
